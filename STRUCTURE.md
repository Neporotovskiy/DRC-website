# Delma Resource Calculator (DRC) - Структура входных данных

---

#### Образец структуры данных

```json
{
    "meta": {
        "type": "Разбруссовка",
        "title": "Дом ДС-20-40 с терассой (John Doe)",
        "vendor": "\"Some Assets Vendor\" Inc.",
        "developer": "Jane Doe",
        "page": {
            "current": 1,
            "total": 1
        },
        "part": {
            "current": 1,
            "total": 1
        }
    },
    "blueprints": [
        {
            "length": 5500,
            "edges": [0, 0],
            "milestones": {
                "468": [1, 1],
                "4768": [1, 1]
            },
            "mark": 57,
            "number": 6,
            "notes": null
        }
    ]
}
```

#### Секция `meta`

Содержит информацию о проекте, поставщике и разработчике структуры данных, а также положение данного документа в проекте.

**Примечания:**

Указанные в данной секции поля несут исключительно информационный характер. Отсутствие какого-либо поля не повлияет на работоспособность алгоритма. В то время как отсутствие данной секции приведет к инвалидации структуры данных.

**Описание полей:**

-   Поле `type` - Указывает тип, описываемый структурой данных.
-   Поле `title` - Указывает имя проекта, к которому относится указанный тип.
-   Поле `vendor` - Указывает поставщика структуры данных.
-   Поле `developer` - Указывает ответственного разработчика структуры данных.
-   Поле `page` - Указывает постраничное положение данного документа в проекте.
-   Поле `part` - Указывает логическое полложение данного документа в проекте.

#### Секция `blueprints`

Содержит информацию об изделиях и их количестве.

**Примечания:**

Некоторые поля интерпретируются справа налево (сам не знаю почему). Подобные поля имеют упоминания направления интерпретации в описании.

Некоторые поля описаны бинарными значениями, где 1 - наличие чего-либо. 0 - отсутствие чего-либо.

Поля, не обозначенные как обязательные в итоговой визуализации не участвуют.

**Опичание полей:**

*   Поле `length` (_обязательное поле_) - Указывает длину изделия (мм)
*   Поле `edges` (_обязательное поле_) - Указывает на наличие торцевых вырезов в изделии. Интерпретируется справа налево.
*   Поле `milestones` (_обязательное поле_) - Указывает на позиции и тип поперечных вырезов на изделии. Ключ описывает горизонтальную позицию выреза (интерпретируется справа налево), а значения определяют сторону с которой будет производиться вырез (интепретируется сверху вниз).
*   Поле `mark` - Указывает маркировку конкретного изделия. Не влияет на работоспособность алгоритма, может быть опущено.
*   Поле `number` (_обязательное поле_) - Указывает необходимое количество данных изделий.
*   Поле `notes` - Указывает какое-либо примечание к данному изделию.
